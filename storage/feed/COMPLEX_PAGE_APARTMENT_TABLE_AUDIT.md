# Аудит таблицы квартир на странице комплекса

**Страница:** `/complex/:slug` (RedesignComplex)  
**Компонент:** ApartmentTable  
**Источник данных:** `GET /api/v1/blocks/{id}/apartments` → complexPageAdapter → mapApartment

---

## 1. Обнаруженные несоответствия

### 1.1 ₽/м² vs Цена/Площадь

**Проблема:** Две квартиры с одинаковыми отображаемыми «Площадь» (24.1 м²) и «Цена» (5.7 млн ₽) показывают разный ₽/м² (238 185 vs 238 520).

**Причина:** 
- `formatPrice` округляет до одного знака (5.65–5.74 млн → «от 5.7 млн»), поэтому разная фактическая цена (5 700 000 vs 5 730 000) не видна.
- ₽/м² вычисляется из фактических значений: `price / area_total`. Разная цена → разный результат.

### 1.2 price_per_meter в бэкенде

- **FeedSyncService:** при синке `price_per_meter` устанавливается в `null` (строка 386).
- **API** возвращает `price_per_meter: null` для большинства квартир.
- **complexPageAdapter** всегда считает: `pricePerMeter = Math.round(price / areaTotal)` — не использует API.
- Итог: отображение ₽/м² корректно и единообразно вычисляется на фронте.

### 1.3 Кухня «0 м²»

- Используется `a.area?.kitchen ?? 0`.
- ApartmentResource отдаёт `area.area_kitchen` из БД.
- В фиде `area_kitchen` может отсутствовать или быть 0 → отображается «0 м²».

### 1.4 ApartmentTable и formatPrice

- Сейчас используется `formatPrice` из `@/redesign/data/mock-data`.
- Нужно перейти на `@/lib/format` для единообразия.

### 1.5 404 по API

- В консоли видны 404 на `/api/v1/blocks/1-y-donskoy-...` и `/blocks/.../apartments`.
- Ограничение маршрута уже исправлено (slug-id).
- Если 404 сохраняется, блок/квартиры отсутствуют в БД.

## 2. Рекомендации

1. **₽/м²:** оставлять вычисление `price / area` на фронте — это даёт логически согласованные данные.
2. **Бэкенд:** при синке заполнять `price_per_meter = price / area_total`, если `area_total > 0`.
3. **Отображение цены:** при одинаковой «округлённой» цене у разных квартир возможна разница в ₽/м² — это ожидаемо.
4. **Кухня:** при отсутствии данных в фиде оставлять «0 м²» или «—».
5. **formatPrice:** использовать `formatPrice` из `@/lib/format` в ApartmentTable.
